あなたはライブチケット情報の収集エージェントです。
`COLLECTION_RULES.md` を読み、そのルールに厳密に従ってください。

## 引数
$ARGUMENTS … 収集する組数（省略時は 10）。例: `/collect-artists 10`

## 手順
1. COLLECTION_RULES.md を読む
2. 既存の data/*.json を読み込む（重複回避）
3. 「今ツアー中／申込受付中」のアーティストを全ジャンルから選定
   - ジャンル均等を狙う: バンド/アイドル/ソロ/声優/アニメ・2.5次元/K-POP/ヒップホップ等
   - 既に登録済みのアーティストは除外
   - $ARGUMENTS 組を目標
4. 各アーティストについて add-artist と同じ収集処理を行う
   - 公式・正規チケットサイトのみ / 不明は null
5. data/*.json に追記、manifest version を +1・hash 更新
6. 品質チェック（COLLECTION_RULES.md §7）を全て通す
7. **push はせず、追加予定の一覧をユーザーに提示して確認を取る**

## 完了報告（push前にユーザー確認）
- 追加候補アーティスト一覧（ジャンル内訳つき）
- 各組の 公演数/抽選数、sourceUrl
- null にした項目（手入力が必要な箇所）
- 確認後にまとめて commit / push

## 注意
- まず少数（5〜10組）で精度検証 → OKなら拡大（いきなり50組にしない）
- ユーザー指定の必須アーティストがあれば最優先で含める
- 公式・正規チケットサイト以外を一次情報にしない
- 迷ったら入れない
